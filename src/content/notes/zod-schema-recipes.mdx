頻出パターンだけを短く持っておくと、画面追加時の判断が速くなります。

## 使い分け

- `z.coerce.*`: フォーム入力や frontmatter の正規化
- `safeParse`: UIでエラーを返したいとき
- `transform`: 表示用の値に寄せるとき

```ts
const articleSchema = z.object({
  title: z.string().min(1),
  tags: z.array(z.string()).default([]),
  date: z.coerce.date()
});
```

複雑な `refine` を増やす前に、入力モデルを分割できないかを先に見ます。
